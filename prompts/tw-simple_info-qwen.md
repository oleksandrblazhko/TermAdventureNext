# tw-simple: Ієрархічна класифікація властивостей ігрового світу

## 1. Сутності гри (Game Entities)

### 1.1 Ієрархія типів сутностей

```
thing (t) — базовий тип для фізичних об'єктів
    |
    +-- container (c) — контейнери (відкриваються, замикаються)
    |       |
    |       +-- oven — спеціальний контейнер (піч/приготування)
    |
    +-- supporter (s) — поверхні для розміщення об'єктів
    |       |
    |       +-- stove — спеціальна поверхня (плита)
    |
    +-- door (d) — двері для з'єднання кімнат
    |
    +-- portable_object (o) — загальні портативні об'єкти
            |
            +-- key (k) — ключі для замків
            |
            +-- food (f) — їстівні предмети

room (r) — кімати (не мають батьківського типу)

player (P) — КОНСТАНТА: сутність гравця
inventory (I) — КОНСТАНТА: інвентар гравця
```

## 2. Властивості сутностей (Predicates/Properties)

### 2.1 Станові властивості (unary predicates)

| Властивість | Типи | Опис |
|-------------|------|------|
| `closed(x)` | c, d | Об'єкт закритий |
| `open(x)` | c, d | Об'єкт відкритий |
| `locked(x)` | c, d | Об'єкт замкнений |
| `edible(f)` | f | Їжа є їстівною |
| `eaten(f)` | f | Їжа була з'їдена |

### 2.2 Просторові відношення (binary/trinary predicates)

| Відношення | Аргументи | Опис |
|------------|-----------|------|
| `at(entity, room)` | (t, r) | Сутність знаходиться у кімнаті |
| `at(P, r)` | (P, r) | Гравець знаходиться у кімнаті |
| `in(entity, container)` | (o, c) | Об'єкт всередині контейнера |
| `in(entity, I)` | (o, I) | Об'єкт в інвентарі гравця |
| `on(entity, supporter)` | (o, s) | Об'єкт на поверхні |
| `link(room1, door, room2)` | (r, d, r) | Двері з'єднують дві кімнати |
| `free(room1, room2)` | (r, r) | Прохід між кімнатами вільний |
| `match(key, door/container)` | (k, d) або (k, c) | Ключ підходить до замка |

### 2.3 Направляючі відношення (binary predicates)

| Відношення | Опис |
|------------|------|
| `north_of(r1, r2)` | Кімната r1 на північ від r2 |
| `south_of(r1, r2)` | Кімната r1 на південь від r2 |
| `east_of(r1, r2)` | Кімната r1 на схід від r2 |
| `west_of(r1, r2)` | Кімната r1 на захід від r2 |

## 3. Операції гравця (Player Actions)

### 3.1 Навігація

| Команда | Дія | Умови |
|---------|-----|-------|
| `go north` | Рух на північ | Кімната з'єднана, прохід вільний |
| `go south` | Рух на південь | Кімната з'єднана, прохід вільний |
| `go east` | Рух на схід | Кімната з'єднана, прохід вільний |
| `go west` | Рух на захід | Кімната з'єднана, прохід вільний |

### 3.2 Взаємодія з контейнерами

| Команда | Дія | Умови |
|---------|-----|-------|
| `open <container>` | Відкрити контейнер | Контейнер закритий, не замкнений |
| `close <container>` | Закрити контейнер | Контейнер відкритий |
| `lock <container> with <key>` | Замкнути контейнер | Ключ підходить, контейнер закритий |
| `unlock <container> with <key>` | Відімкнути контейнер | Ключ підходить, контейнер замкнений |
| `take <object> from <container>` | Взяти об'єкт з контейнера | Контейнер відкритий, об'єкт всередині |
| `insert <object> into <container>` | Вставити об'єкт у контейнер | Контейнер відкритий |

### 3.3 Взаємодія з дверима

| Команда | Дія | Умови |
|---------|-----|-------|
| `open <door>` | Відкрити двері | Двері закриті, не замкнені |
| `close <door>` | Закрити двері | Двері відкриті |
| `lock <door> with <key>` | Замкнути двері | Ключ підходить, двері закриті |
| `unlock <door> with <key>` | Відімкнути двері | Ключ підходить, двері замкнені |
| `examine <door>` | Оглянути двері | — |

### 3.4 Взаємодія з поверхнями

| Команда | Дія | Умови |
|---------|-----|-------|
| `put <object> on <supporter>` | Покласти об'єкт на поверхню | Об'єкт в інвентарі |
| `take <object> from <supporter>` | Взяти об'єкт з поверхні | Об'єкт на поверхні |

### 3.5 Взаємодія з їжею

| Команда | Дія | Умови |
|---------|-----|-------|
| `eat <food>` | З'їсти їжу | Їжа в інвентарі, їстівна |
| `cook <food>` | Приготувати їжу | Їжа на плиті (stove) |

### 3.6 Загальні дії

| Команда | Дія |
|---------|-----|
| `take <object>` | Взяти об'єкт з кімнати |
| `drop <object>` | Випустити об'єкт у кімнаті |
| `inventory` | Перевірити інвентар |
| `look` | Оглянутися |
| `examine <object>` | Оглянути об'єкт |

## 4. Структура квесту (Quest Structure)

### 4.1 Послідовність дій для перемоги (sparse rewards)

1. **Знайти ключ**: `open chest drawer` → `take old key from chest drawer`
2. **Відімкнути двері**: `unlock wooden door with old key`
3. **Відкрити двері**: `open wooden door`
4. **Вийти з кімнати**: `go east`
5. **Знайти їжу**: `open refrigerator` → `take apple from refrigerator`
6. **Приготувати їжу**: `put apple on stove`

### 4.2 Умови перемоги (Win Conditions)

Квест завершується успішно, коли виконано:
- `on(food, stove)` — їжа розміщена на плиті

### 4.3 Умови поразки (Fail Conditions)

Квест завершується невдачею, якщо:
- `eaten(food)` — їжа була з'їдена (замість того, щоб бути приготованою)

## 5. Варіанти налаштувань (Settings)

### 5.1 Частота винагород (--rewards)

| Режим | Опис |
|-------|------|
| `dense` | Багато дрібних підквестів (відкрити контейнер, взяти ключ, відімкнути двері, тощо) |
| `balanced` | Середня кількість підквестів (вихід з кімнати, пошук їжі, приготування) |
| `sparse` | Мінімальна кількість підквестів (тільки приготування їжі) |

### 5.2 Опис цілі (--goal)

| Режим | Опис |
|-------|------|
| `detailed` | Детальний опис цілі |
| `brief` | Короткий опис (напр., "The dinner is almost ready! It's only missing a grilled apple.") |
| `none` | Без опису цілі |

## 6. Обмеження та правила (Constraints)

### 6.1 Взаємовиключні стани
- `open` та `closed` — взаємовиключні
- `open` та `locked` — взаємовиключні
- `closed` та `locked` — взаємовиключні

### 6.2 Обмеження розміщення об'єктів
- Об'єкт може бути лише в одному місці: інвентар, контейнер, поверхня або кімната
- Гравець може бути лише в одній кімнаті
- Ключ відмикає лише один замок (one-to-one mapping)

### 6.3 Обмеження дверей
- Двері з'єднують максимум дві кімнати
- Кімната має максимум 4 виходи
