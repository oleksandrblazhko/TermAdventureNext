# Список команд Bash-оболонки, які використовуються у навчальній лабораторній роботі за темою "Основи роботи з Unix-подібною файловою системою"
1) ls каталог (list – список) - переглянути вміст каталогу за вказаною назвою або поточного каталогу, якщо назва не вказана
2) cd каталог - перейти до іншого каталогу або домашнього каталогу користувача, якщо назва каталогу не вказана; для переходу до каталогу на рівень вище (батьківський каталог) у назві каталогу вказуються символи дві крапки .. , наприклад, cd ..
3) cat файл – вивести вміст файлу на екран; використовується для читання стану дверей (`cat door.state`), перевірки вмісту предметів, читання коротких записок.
4) less файл - переглянути вміст файлу (для завершення перегляду файлу використовується клавіша q); використовується для детального огляду довгих предметів (книги, інструкції, логи).
5) pwd – переглянути повний шлях від кореневого каталогу до поточного робочого каталогу
6) mv джерело призначення (move) – перемістити файл; **основна команда взаємодії з предметами**: **взяти предмет** (`mv контейнер/предмет ~/`), **покласти предмет** (`mv ~/предмет поверхня/`), **перекласти між контейнерами** (`mv скриня/ключ кишеня/`). Предмет зникає з джерела і з'являється в призначенні – як у реальному світі.
7) pushd каталог - перейти до іншого каталогу за вказаною назвою з можливістю швидкого повернення до нього, завдяки тому, що команда формує власний стек із назвами каталогів та виводить його на екран; виконання команди без параметру перемикає між двома верхніми каталогами у стеку
8) popd - повернутися до каталогу, з якого виконано переміщення після останньої команди pushd, взявши назву каталогу з вершини стеку
9) rm файл (remove) – видалити файл; використовується для **відчинення контейнерів** (`rm .closed`), **видалення тимчасових файлів**, **знищення предметів** (напр. `rm apple` = з'їсти яблуко).
10) touch файл – створити порожній файл або оновити час модифікації; використовується для **зачинення контейнерів** (`touch .closed`), **позначення відчиненого стану** (`touch .open`), **створення прапорців** (`touch win_condition`).
11) mkdir каталог (make directory) – створити директорію; використовується для **підготовки ігрового середовища** (precmd), створення контейнерів та кімнат.
12) echo $змінна_ середовища - переглянути значення змінної оточуючого середовища ОС, першим прикладом, якої є змінна $PATH із переліком каталогів, необхідних для швидкого пошуку файлів, які можуть виконуватися в командному рядку без вказування повного шляху до файлу, при цьому каталоги розділяються символом двокрапка ':'
13) date – переглянути поточну дату із визначенням окремих показників часу, наприклад, команда: `date +'%e день %m місяця %G року'` – виведе день (%e), місяць (%m) та рік (%G), але більше форматів можна отримати через виклик date –help
14) alias псевдонім="команда з прапорцями" (псевдонім) - створити псевдонім (скорочену назву) для Bash-команди, який буде пам'ятати прапорці команди, 
що дозволяє спростити майбутні дії користувача
15) whoami - переглянути ім'я користувача, який запустив оболонку
16) cp джерело призначення (copy) – копіювати файл або директорію; використовується рідко – коли предмет треба **розмножити** (напр. зробити копію ключа для іншого гравця). У більшості випадків замість cp слід використовувати mv.
17) uname – надрукувати інформацію про комп'ютер, використовуючи прапорці: n – мережева назва комп'ютера, m – апаратна платформа комп'ютера, s – назва ядра ОС, a – повний опис.
