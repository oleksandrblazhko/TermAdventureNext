# Таблиця мапінгу дій гравця TextWorld на Bash-команди

| ID ієрархії | Parent-сутність (батьківська) | Child-сутність (дочірня) | Назва дії гравця (Player Action) | Назва Bash-команди | Рівень відповідності | Пояснення щодо відповідності дій Bash-команди та дій гравця |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **1.** | **Player (Гравець)** | - | - | - | - | - |
| 1.1. | Player | Room (Кімната) | `look` (оглянути) | `ls` | > 75% | `ls` дозволяє переглянути вміст поточного каталогу (кімнати), що відповідає огляду кімнати. |
| 1.2. | Player | Room | `go <direction>` (йти <напрямок>) | `cd` | > 75% | `cd` використовується для переміщення між каталогами (кімнатами) у файловій системі. |
| 1.3. | Player | Inventory (Інвентар) | `inventory` (інвентар) | `ls ~` | > 75% | `ls ~` (або `ls $HOME`) показує вміст домашнього каталогу користувача, який може слугувати інвентарем. |
| **2.** | **Room (Кімната)** | - | - | - | - | - |
| 2.1. | Room | Object (Об'єкт) | `examine <object>` (оглянути <об'єкт>) | `cat` або `less` | > 75% | `cat` або `less` дозволяють переглядати вміст файлів (опис об'єкта). `less` краще для великих об'єктів. |
| **3.** | **Inventory (Інвентар)** | - | - | - | - | - |
| 3.1. | Inventory | Object | `examine <object>` (оглянути <об'єкт>) | `cat` або `less` | > 75% | Аналогічно огляду об'єкта в кімнаті, `cat` або `less` для перегляду вмісту файлу-об'єкта. |
| 3.2. | Inventory | Object | `drop <object>` (викинути <об'єкт>) | `mv` | > 75% | `mv ~/object .` перемістить об'єкт з інвентаря (домашнього каталогу) в поточний каталог (кімнату). |
| **4.** | **Thing (Річ)** | - | - | - | - | - |
| 4.1. | Thing | Itself | `examine <thing>` (оглянути <річ>) | `cat` або `less` | > 75% | Загальний принцип огляду сутності через перегляд вмісту файлу, що її представляє. |
| **5.** | **Container (Контейнер)** | - | - | - | - | - |
| 5.1. | Container | Itself | `open <container>` (відкрити <контейнер>) | `rm .closed` | > 75% | Видалення файлу `.closed` у каталозі-контейнері символізує його відкриття. |
| 5.2. | Container | Itself | `close <container>` (закрити <контейнер>) | `touch .closed` | > 75% | Створення файлу `.closed` у каталозі-контейнері символізує його закриття. |
| 5.3. | Container | Key (Ключ) | `unlock <container> with <key>` (відімкнути <контейнер> ключем) | `mv key .` та `rm .locked` | приблизно 50% | Розблокування може включати переміщення ключа та видалення файлу `.locked`. |
| 5.4. | Container | Key | `lock <container> with <key>` (замкнути <контейнер> ключем) | `mv key .` та `touch .locked` | приблизно 50% | Блокування може включати переміщення ключа та створення файлу `.locked`. |
| 5.5. | Container | Object (Об'єкт) | `take <object> from <container>` (взяти <об'єкт> з <контейнера>) | `mv container/object ~/` | > 75% | Переміщення файлу-об'єкта з каталогу-контейнера до домашнього каталогу. |
| 5.6. | Container | Object | `insert <object> into <container>` (покласти <об'єкт> в <контейнер>) | `mv ~/object container/` | > 75% | Переміщення файлу-об'єкта з домашнього каталогу до каталогу-контейнера. |
| **6.** | **Supporter (Поверхня)** | - | - | - | - | - |
| 6.1. | Supporter | Object | `take <object> from <supporter>` (взяти <об'єкт> з <поверхні>) | `mv supporter/object ~/` | > 75% | Переміщення файлу-об'єкта з каталогу-поверхні до домашнього каталогу. |
| 6.2. | Supporter | Object | `put <object> on <supporter>` (покласти <об'єкт> на <поверхню>) | `mv ~/object supporter/` | > 75% | Переміщення файлу-об'єкта з домашнього каталогу до каталогу-поверхні. |
| **7.** | **Door (Двері)** | - | - | - | - | - |
| 7.1. | Door | Itself | `open <door>` (відкрити <двері>) | `rm door/.closed` | > 75% | Видалення файлу `.closed` у каталозі-двері символізує її відкриття. |
| 7.2. | Door | Itself | `close <door>` (закрити <двері>) | `touch door/.closed` | > 75% | Створення файлу `.closed` у каталозі-двері символізує її закриття. |
| 7.3. | Door | Key | `unlock <door> with <key>` (відімкнути <двері> ключем) | `mv key door/` та `rm door/.locked` | приблизно 50% | Розблокування може включати переміщення ключа та видалення файлу `.locked`. |
| 7.4. | Door | Key | `lock <door> with <key>` (замкнути <двері> ключем) | `mv key door/` та `touch door/.locked` | приблизно 50% | Блокування може включати переміщення ключа та створення файлу `.locked`. |
| **8.** | **Food (Їжа)** | - | - | - | - | - |
| 8.1. | Food | Itself | `eat <food>` (з'їсти <їжу>) | `rm food_item` | > 75% | Видалення файлу їжі символізує її з'їдання (зникнення). |
| **9.** | **Object (Об'єкт)** | - | - | - | - | - |
| 9.1. | Object | Itself (в кімнаті) | `take <object>` (взяти <об'єкт>) | `mv object ~/` | > 75% | Переміщення файлу-об'єкта з поточного каталогу до домашнього каталогу. |

**Bash команди, які не увійшли до таблиці:**

*   `pushd`, `popd`: Ці команди пов'язані з навігацією, але їхня функціональність "стеку" каталогів не має прямого аналога в ігрових діях "go <direction>". Хоча `cd` добре справляється з переміщенням, `pushd/popd` дозволяють більш складні сценарії збереження та відновлення позиції, що не є прямою дією гравця, а скоріше механікою. Рівень відповідності менше 25% для прямої дії.
*   `echo`, `date`, `alias`, `whoami`, `cp`, `uname`: Ці команди виконують службові або інформаційні функції, які не відповідають прямим ігровим діям (огляд, взаємодія з об'єктами, переміщення). Хоча `echo` може бути використаний для створення вмісту файлів (як для `less`), це не є прямою "дією гравця" в контексті гри TextWorld. `cp` може бути використаний для "розмноження предметів", але в більшості випадків `mv` є більш відповідним для "взяти/покласти".
    *   `echo`: менше 25% (виведення тексту, але не пряма дія)
    *   `date`: менше 25% (інформація про час, не ігрова дія)
    *   `alias`: менше 25% (налаштування оболонки, не ігрова дія)
    *   `whoami`: менше 25% (інформація про користувача, не ігрова дія)
    *   `cp`: приблизно 50% (копіювання предметів, але `mv` частіше використовується для "взяти/покласти")
    *   `uname`: менше 25% (системна інформація, не ігрова дія)
