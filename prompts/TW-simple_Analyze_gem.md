# Аналіз сутностей, станів та дій у TextWorld

## 1. Сутності та Ієрархія

На основі аналізу файлів `KB` (бази знань) ігрових JSON-файлів, можна виділити наступну ієрархію сутностей:

- **`t` (thing)** - річ: базовий клас для всіх об'єктів у грі.
  - **`o` (object)** - об'єкт: переносні предмети.
    - **`f` (food)** - їжа: їстівні об'єкти.
    - **`k` (key)** - ключ: об'єкт для замикання/відмикання.
  - **`s` (supporter)** - поверхня: місце, де можуть лежати об'єкти (стіл, ліжко). Нерухомі.
  - **`c` (container)** - контейнер: може містити об'єкти, бути відкритим/закритим/замкненим. Нерухомий.
  - **`d` (door)** - двері: з'єднують кімнати, можуть бути відкритими/закритими/замкненими.
- **`r` (room)** - кімната: локація, де знаходиться гравець та об'єкти.
- **`P` (Player)** - гравець: дійова особа, якою керує користувач.
- **`I` (Inventory)** - інвентар: "контейнер" гравця для переносних об'єктів.

| 1.2. | Player | Room | `go <direction>` (йти <напрямок>) | Переміщення гравця між кімнатами у вказаному напрямку (north, south, east, west). | `go east` |
| 1.3. | Player | Inventory (Інвентар) | `inventory` (інвентар) | Перевірити вміст інвентарю гравця. | `You are carrying: an apple.` |

| 2.1. | Room | Object (Об'єкт) | `examine <object>` (оглянути <об'єкт>) | Отримати детальний опис об'єкта, який знаходиться в кімнаті. | `examine note` |
| **3.** | **Inventory (Інвентар)** | - | - | - | - |

| 3.2. | Inventory | Object | `drop <object>` (викинути <об'єкт>) | Викидає об'єкт з інвентарю в поточну кімнату. | `drop apple` |
| **4.** | **Thing (Річ)** | - | - | - | - |
