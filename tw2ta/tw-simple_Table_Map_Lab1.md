## Bash команди, не включені до таблиці та причини їх відсутності:

Ці команди Bash не були включені до таблиці зіставлення, оскільки вони не мають прямої, аналогічної дії гравця, описаної в розділі "Можливі операції гравця" гри `tw-simple` або використовуються переважно для ігрової логіки/налаштування, а не для прямої взаємодії з гравцем.

*   **`pwd`**: Хоча вона показує поточне місце розташування (кімнату), дія `look`, зіставлена з `ls -la`, надає більш вичерпний опис вмісту кімнати, що більше відповідає взаємодії гравця. `pwd` сама по собі не є окремою дією гравця в ігровому оповіданні.
*   **`pushd` / `popd`**: Ці команди керують історією каталогів для навігації. Рух у грі `tw-simple` (`go north/south/east/west`) простіший і не передбачає історію кімнат на основі стеку для гравців.
*   **`test`**: Ця команда в основному використовується для умов сценаріїв у внутрішній частині гри (файли `.ta`) для перевірки станів (наприклад, чи існує ключ). Це внутрішній механізм ігрової логіки, а не дія гравця.
*   **`mkdir`**: Використовується для створення каталогів, що в контексті гри описується як "підготовка ігрового середовища" або створення контейнерів/кімнат. Це адміністративні дії або дії з налаштування, а не типові внутрішньоігрові команди гравця.
*   **`echo`**: Використовується для відображення значень змінних середовища. Гра не описує жодних дій гравця, які безпосередньо включають відображення системних змінних.
*   **`date`**: Відображає поточну дату та час. Ігровий світ `tw-simple` не містить механізмів, заснованих на часі, або дій гравця, пов'язаних з перевіркою дати.
*   **`alias`**: Використовується для створення ярликів команд. Це конфігурація на рівні користувача для середовища оболонки, а не дія в рамках ігрового оповідання.
*   **`whoami`**: Відображає поточне ім'я користувача. У `tw-simple` немає дій гравця, пов'язаних з ідентифікацією системного користувача гравця.
*   **`cp`**: Описана як така, що рідко використовується для дублювання предметів. Основні дії з маніпуляції предметами (взяття, скидання, переміщення предметів) покриваються `mv`. `cp` представляє дуже специфічну, менш поширену дію, не центральну для перелічених взаємодій у грі.
*   **`uname`**: Відображає системну інформацію. Гра не включає дії гравця для запиту базових відомостей про операційну систему.
